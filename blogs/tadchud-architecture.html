We just finished to design and considered about technology stack. I want to explain why we decide to use each tools. Some of them you might not hear about, do not use it, or never implement on your projects.

Techology stack

Frameworks

Next.js(React)

Actually we don't need server side rendering but we're lazy people to use React Router. So Next.js is the best solution to automatically generate routes. That's it~~.
We decided to use two UI libraries
  • Chakra-UI
  • Mantine-UI

    • Graphql

    It's a Query language for API that we can select specifically fields that we need and it has only one endpoint. In contrast REST API has over-fetching, under-fetching, and multiple endpoints to get all data.

    Sample Graphql query

    {
  product {
    name
    height
    variants
  }
}

    Fastify

    Almost the same as ExpressJS but better 😎😎😎. Well, there is no everything perfect, so we made some internal plugins to fix some issues from fastify and more compatiable with Graphql. Fastify also faster than others Nodejs frameworks that we know.

    Databases

    PostgreSQL

    We decide to use PostgreSQL because it's a popular and powerful object-relational database. And we have experienced to use in many projects(Small and Medium sizes). it has good performance and comprehensively support unstructured data eg. JSON, JSONB which we have to use to store customization data because we still don't have much experience with 3D/2D customization tool so we're not sure how many fields and what data type of each so unstructured data is very suitable.

    Typesense Search engine

    You might hear or use Elasticsearch, Agolia, Apache solr which are very popular and many companies use them. As you know we are Tech enthusiasts persons so we're looking forward to new tools that we never use before. As we have been tried Typesense, it very powerful, and easy to implement and set up. We hopefully will not change to others later. 🤣🤣

    Redis

    Of course your guessing is right. Redis is a in-memory database for caching data to prevent our PostgreSQL database get hurt from every request. Keeps the dataset in memory for fast access, but can also persist all writes to permanent. We also implement it on top of Event driven architecture as publisher and subscriber engine to communicate internal services.
